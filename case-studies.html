Case Study 01

Maya HDA Tool

Python / Houdini Engine / SVN

Overview

A Maya-integrated tool for loading, managing, and updating Houdini Digital Assets (HDAs) directly inside Maya via Houdini Engine. The tool bridges the gap between Houdini's procedural asset creation and Maya-based art production — handling license management and SVN version control automatically so artists can stay focused on their work.

HDA Tool v0.2.1 — asset browser with SVN update controls

The interface surfaces available HDAs — including EdgeDecal, FractureCut, and VertexHeatmap — with thumbnail previews, descriptions, and last-updated timestamps. Each asset has a one-click update button to pull the latest version from SVN, and an info button linking directly to its documentation.

Problem

Artists working in Maya needed access to Houdini Digital Assets for procedural geometry, destruction setups, and vertex data, but loading, updating, and managing HDA versions was a manual and technical process. Houdini Engine license management added extra friction, and keeping assets in sync with SVN required leaving the Maya context entirely.

Key Features

HDA Loading & Management

Clean Maya interface to browse, load, and manage HDAs without needing to understand the Houdini Engine API. Thumbnail previews and descriptions make assets immediately identifiable.

Houdini Engine License Management

Automatically handles Houdini Engine license acquisition and release via a config toggle. The 0/1 license indicator gives instant visibility into license state — removing a common source of confusion for Maya-primary artists.

SVN Version Control Integration

Built-in SVN calls allow the tool to check for and pull HDA updates directly from within Maya. The last-changed date confirms asset currency at a glance — no manual SVN operations required.

Documentation Links

Each loaded asset connects to its documentation page, giving artists immediate access to usage instructions without leaving Maya.

Tech Stack

Python Maya API Houdini Engine SVN HDAs

Impact

  • Enabled Maya-primary artists to use procedural HDA assets without Houdini knowledge
  • Automated Houdini Engine license management, eliminating a frequent pain point
  • SVN integration kept assets in sync without requiring manual version control steps
  • Documentation links reduced support requests and improved artist independence

Case Study 02

Maya Tools for Environment Artists

PyQt5 / PySide2 / Python

Overview

A comprehensive Maya toolset — Environment Art Tools v1.2 — built to reduce manual clicks and automate repetitive tasks for the environment art team. The suite covers the full environment artist workflow in a single tabbed interface: model validation, material management, asset publishing, Havok physics setup, QA checks, and configurable settings.

Help & Documentation

Built-in Help menu — documentation (F1) and developer contact

A built-in Help menu provides instant access to documentation and a direct contact link to the tool developer — reducing support overhead and making the toolset self-contained for new team members.

Model Tab

Model tab — full modelling workflow utilities

The primary workspace. Artists name assets, flag them as new or destructible, and access grouped utilities covering the full modelling workflow — Reset Mesh, Toggle Pivot, Show/Hide BSP, UV creation and swapping, file references, and one-click export to Mesh, Triangle Mesh, or VFX format.

Materials Tab

Materials tab — full material list with create, navigate, and delete controls

Full material management panel. Artists create and link materials by type, delete unused shaders, toggle vertex colour display, paint by material, and manage the complete material list per asset — with colour swatches and per-entry controls.

Asset Manager Tab

Asset Manager tab — detail-set management and export

Centralised detail-set management. Supports Update Scene, Update Selected, Export Assembly Definition, Export Source, and Rename Asset — keeping environment assets in sync across the pipeline without leaving Maya.

Havok Tab

Havok tab — physics setup with mass, strength, and hardpoint controls

Automates Havok physics collision setup. Artists configure mass and strength values and apply them to selections, create physics layers, and add hardpoints — covering a workflow that was previously manual and error-prone.

QA Tab

QA tab — geometry validation before submission

Quality assurance utilities that validate assets before submission. Show Intersections, Check Scene Translations, and Select Borders surface geometry issues early — catching problems before they reach integration.

Settings Tab

Settings tab — configurable paths and UI preferences

Configurable project paths for source files, the Artmain directory, and the Asset Manager directory. Export and UI preferences allow the toolset to adapt to different project setups and personal workflows.

Tech Stack

Python PyQt5 PySide2 Maya API Havok

Impact

  • Reduced manual steps across the environment team's full asset workflow
  • Standardised QA and export processes, cutting downstream pipeline errors
  • Single tabbed interface covering model, material, asset, physics, and QA stages
  • Configurable settings allowed the tool to adapt across projects and team members
  • Built-in help and documentation lowered onboarding time for new artists
